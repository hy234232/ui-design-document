> **Status:** `Draft` · **Last updated:** 2026-06-19  
> **Owner:** Design System Team

## 1. Overview

PC 환경에서 모든 모달(Dialog/Modal)이 공통으로 따르는 **높이 규칙**을 정의한다.  
모달 내용의 양과 무관하게 화면을 가리지 않으면서도 너무 납작하지 않도록, **최소 높이는 절대값**으로, **최대 높이는 기기(뷰포트) 기준 상대값**으로 제어한다.

> 이 문서는 특정 모달 1개가 아니라 **모달 컴포넌트 전체에 적용되는 높이 공통 규칙**이다.  
> 너비/타이틀/버튼 등 다른 스펙은 별도 모달 컴포넌트 문서를 따른다.

---

## 2. Anatomy

높이 규칙은 모달을 세 영역으로 나눠서 적용한다. 스크롤은 **텍스트(본문) 영역에서만** 발생한다.

```
┌───────────────────────────────┐
│  Header   (타이틀 + 닫기 X)    │  ← 고정
├───────────────────────────────┤
│                               │
│  Text / Body  (스크롤 영역)    │  ← 내용 많으면 이 영역만 스크롤
│                               │
├───────────────────────────────┤
│  Footer   (액션 버튼)          │  ← 고정
└───────────────────────────────┘
```

| # | Element | Required | 높이 동작 | Figma Layer Name |
|---|---------|----------|-----------|------------------|
| 1 | Header | ✅ | 고정 (콘텐츠 높이) | `header` |
| 2 | Text / Body | ✅ | 가변 + 스크롤 | `body` / `text-area` |
| 3 | Footer | 조건부 | 고정 (콘텐츠 높이) | `footer` |

---

## 3. Height Rules

### 3-1. 최소 높이 (Minimum) — 절대값

콘텐츠가 적어도 아래 값보다 작아지지 않는다. (px 고정)

| 대상 | 최소 높이 | 설명 |
|------|-----------|------|
| 모달 전체 | `240px` | 모달 컨테이너 전체 높이의 하한 |
| 텍스트(본문) 영역 | `70px` | Header/Footer를 제외한 본문 영역의 하한 |

> **예시:** 로그아웃 확인처럼 본문이 한 줄("로그아웃 하시겠어요?")뿐이어도, 본문 영역은 `70px` 이상을 유지하고 모달 전체는 `240px` 이상을 유지한다.

### 3-2. 최대 높이 (Maximum) — 상대값

| 대상 | 최대 높이 | 초과 시 동작 |
|------|-----------|--------------|
| 모달 전체 | 기기(뷰포트) 높이의 `90%` | **텍스트(본문) 영역에 스크롤** 처리 |

- 모달 전체 높이는 뷰포트 높이의 `90%`를 **절대 넘기지 않는다.**
- 내용이 길어 `90%`를 초과하게 되면, **Header/Footer는 고정**하고 **본문(text-area)만 내부 스크롤**한다.
- 모달 자체가 화면 밖으로 잘리거나 `90%`를 넘기는 형태는 **금지**한다. (아래 ❌ 예시)

## 5. Implementation Notes (Frontend)

본문 영역만 스크롤되도록 **3분할 + flex 레이아웃**으로 구현한다.

```css
.modal {
  min-height: 240px;
  max-height: 90vh;            /* 뷰포트 높이 90% 상한 */
  display: flex;
  flex-direction: column;
}

.modal__header,
.modal__footer {
  flex: 0 0 auto;              /* 고정 */
}

.modal__body {
  flex: 1 1 auto;
  min-height: 70px;            /* 본문 영역 최소 높이 */
  overflow-y: auto;           /* 초과분만 이 영역에서 스크롤 */
}
```

| 항목 | 값 | 비고 |
|------|----|------|
| `min-height` (전체) | `240px` | 절대값 |
| `min-height` (본문) | `70px` | 절대값 |
| `max-height` (전체) | `90vh` | 상대값, 뷰포트 기준 |
| 스크롤 컨테이너 | `.modal__body` | Header/Footer는 스크롤 영역 밖 |

> `90%` 기준은 모달이 떠 있는 **뷰포트(브라우저 창)** 높이다. 모바일 규격은 본 문서(PC) 범위 밖이며 별도 정의한다.

## 6. Do & Don't

| ✅ Do | ❌ Don't |
|-------|----------|
| 본문이 짧아도 전체 `240px`/본문 `70px` 유지 | 콘텐츠에 맞춰 모달을 `240px`보다 작게 축소 |
| 길어지면 본문 영역만 내부 스크롤 | 모달 전체를 페이지처럼 스크롤 |
| 전체 높이를 `90vh` 이내로 고정 | 뷰포트 `90%`를 넘겨 화면을 꽉 채우거나 잘림 |
| Header/Footer는 항상 보이게 고정 | 스크롤 시 타이틀/액션 버튼이 같이 밀려 사라짐 |
