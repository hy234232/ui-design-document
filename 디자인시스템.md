# 디자인시스템 정의 (Spec-of-Record)

> **목적**: 모든 화면·기능에 일관되게 적용되는 컴포넌트 동작·상태·문구 규칙을 한 곳에서 관리.
> 이 파일은 **server.js가 매 분석 요청마다 자동으로 읽어** 프롬프트에 주입합니다.
> 따라서 여기에 정의한 내용이 곧 생성되는 기능 명세의 FE/BE 할일 문구가 됩니다.
>
> **편집 규칙**:
> - 규칙은 짧고 결정적으로 작성. ("3초 뒤 fade-out" O / "적당히 사라짐" X)
> - 컴포넌트 단위로 묶기. 한 컴포넌트의 모든 상태(default·hover·active·disabled·error)를 한 섹션에 모음.
> - 새 컴포넌트 추가 시 이 파일에 정의한 다음 사용. 정의되지 않은 컴포넌트는 LLM 추론에 맡겨짐.
>
> **컴포넌트별 md 링크 컨벤션 (필수 형식):**
> 각 컴포넌트 섹션 최상단에 한 줄로:
> ```
> - **컴포넌트 md (GitHub)**: [컴포넌트명 컴포넌트 md](GitHub URL)
> ```
> server.js는 이 형식을 인식해서 해당 컴포넌트를 사용하는 기능 명세 결과에 `docLink`(클릭 가능한 링크)를 자동으로 첨부합니다.
> GitHub URL은 컴포넌트 md 파일이 커밋된 후의 raw/blob 링크. 아직 푸시 전이면 `PLACEHOLDER` 로 둬도 동작은 함(링크는 동작 안 함).

## 토스트(Toast) · 스낵바(Snackbar)

- **컴포넌트 md (GitHub)**: [Toast Notification 컴포넌트 md](https://github.com/PLACEHOLDER_OWNER/PLACEHOLDER_REPO/blob/main/design-system/toast-notification.md)
  - 로컬 사본: [design-system/toast-notification.md](design-system/toast-notification.md)
- **표시 동작**: fade-in 후 **3초 뒤 자동으로 fade-out** 으로 사라짐.
- **위치**: 화면 하단 중앙 또는 우측 하단. 한 번에 하나만 표시(같은 메시지 중복 차단).
- **유형별 컬러**: 성공=초록, 정보=파랑, 경고=노랑, 실패=빨강.
- **닫기 버튼**: 우측에 X 아이콘. 사용자가 직접 닫을 수도 있음.
- **사용 시점**: 액션 성공·실패·정보 알림. 사용자 입력을 막지 않음(non-blocking).
- **FE 표준 문구**: "토스트 알림 표시 — fade-in 후 3초 뒤 fade-out으로 자동 사라짐"

---

## 모달(Modal) · 다이얼로그(Dialog) · 팝업(Popup)

- **구조**: 화면 전체를 어둡게 덮는 **딤(dim) 오버레이** + 중앙에 흰색 라운드 카드.
- **딤 처리**: 검정 40% 투명도. 모달 닫을 때 딤도 함께 제거.
- **닫기 동작 3종 (예외 없이 모두 구현)**:
  1. 모달 내부의 **취소/닫기/아니오** 버튼 (화면에 실제 보이는 라벨 사용)
  2. 모달 우상단 **X 아이콘** 버튼
  3. 모달 외 영역(**딤 배경**) 클릭
- **변경 사항 롤백**: 입력/변경 중 내용이 있으면 닫기 시 폐기 확인 또는 자동 롤백.
- **포커스 트랩**: 모달 내부 인터랙티브 요소에만 포커스 이동. ESC 키로도 닫기.
- **폼 검증**: 모달 안 폼은 저장 시 일괄 검증, 실패 시 모달 유지 + 에러 표시.
- **FE 표준 문구**: "모달 표시/딤 오버레이 + (취소/닫기/아니오) 버튼·X 버튼·딤 배경 클릭으로 닫기 + 닫을 때 딤 배경 해제 + 필요 시 변경 롤백 + 포커스 트랩·폼 검증"

---

## 인풋(Input) · 텍스트필드(TextField)

- **상태**: default · focus · filled · error · disabled (읽기 전용).
- **placeholder**: 입력 예시·형식 안내. 입력 시작 시 사라짐.
- **포커스**: 보라 보더(2px) + 약한 그림자.
- **에러**: 빨간 보더 + 인풋 하단에 빨간 헬퍼 텍스트 ("이메일 형식에 맞지 않아요").
- **비활성(disabled/readonly)**: 회색 배경 + 회색 텍스트, 클릭/타이핑 차단. **기능 명세에는 "읽기 전용으로 표시"로 명시**.
- **글자수 카운트**: 최대 길이 제한 있는 인풋은 우측 하단에 `N/MAX` 카운트 표시, 초과 시 입력 차단.
- **검증**: ① 빈값 ② 형식(이메일·전화·날짜) ③ 서버 검증 ④ 네트워크 오류 4단계.

---

## 셀렉트(Select) · 드롭다운(Dropdown) · 콤보박스(Combobox)

- **트리거**: 인풋 형태 + 우측 chevron(▼) 아이콘.
- **드롭다운 패널**: 트리거 아래 펼침, 외부 클릭 시 닫힘. ESC로도 닫힘.
- **선택지 호버**: 연한 보라 배경.
- **선택 후**: 트리거에 선택값 표시, 패널 닫힘.
- **비활성**: 회색 + 클릭 차단.

---

## 버튼(Button)

- **유형**: primary(보라) · secondary(회색) · text(투명) · danger(빨강) · disabled(흐림).
- **상태**: default · hover(살짝 어둡게) · active(눌림 효과) · loading(스피너) · disabled.
- **사이즈**: H44(large) · H32(medium) · H24(small) — 기본 H32.
- **라운드**: 8px (small은 6px).
- **로딩 상태**: 클릭 후 응답 대기 중에는 스피너 + 클릭 차단.

---

## 체크박스(Checkbox) · 라디오(Radio) · 토글(Switch)

- **체크박스**: 다중 선택. 라벨 클릭으로도 토글.
- **라디오**: 단일 선택. 그룹 안에서 하나만.
- **토글**: ON/OFF 즉시 적용 (저장 버튼 없음). 변경 시 토스트로 결과 안내.

---

## 테이블(Table) · 목록(List)

- **컬럼 헤더**: 굵게, 좌측 정렬(텍스트)·우측 정렬(숫자).
- **행 호버**: 연한 회색 배경.
- **빈 결과**: "결과가 없어요" + 안내 아이콘.
- **페이징 방식 미명시 시 기본값**: **무한 스크롤(infinite scroll)** — 스크롤 끝 도달 시 다음 페이지 lazy load, 로딩 인디케이터 표시.
- **명시적 페이지네이션이 있으면** 페이지 기반 방식 따름(무한 스크롤 가정 금지).

---

## 페이지네이션(Pagination)

- 컴포넌트: 이전 · 페이지 번호(1~N) · 다음 · 끝.
- 현재 페이지는 보라 강조.
- 페이지당 개수 셀렉트(10/20/50)가 있으면 그 옵션 따름.

---

## 인사이트 알럿(Insight Alert)

- **위치**: 페이지 상단(타이틀 아래) 또는 섹션 위.
- **스타일**: 연보라 배경(`#F4F1FE`) + 좌측 ✨ 스파클 아이콘 + 본문 텍스트.
- **본문**: 데이터 기반 자동 생성 문구(예: "30대 여성이 가장 많이 결제했어요"). 데이터 변경 시 재생성.
- **FE 표준 문구**: "인사이트 알럿 — 자동 생성 문구 표시 + 데이터 변경 시 재생성"

---

## 차트(Chart) · 그래프

- **유형**: 막대·라인·도넛·영역·히트맵.
- **호버 툴팁**: 흰 배경 카드 + 약한 그림자 + 데이터 라벨·값.
- **범례**: 차트 좌측 하단. 컬러 점(●) + 항목명.
- **단위**: 차트 우측 하단에 "단위: 원/회/명/%/도" 표기.
- **빈 데이터**: "데이터가 없어요" 안내.

---

## 뱃지(Badge) · 태그(Tag) · 칩(Chip)

- **상태 뱃지**: 성공=초록 · 경고=주황 · 실패=빨강 · 정보=파랑.
- **카테고리 태그**: 회색 배경 + 라운드 4px.
- **선택 가능한 칩**: 클릭으로 토글, 활성 시 보라 배경.

---

## 헤더(GNB) · 사이드바(LNB)

- **GNB 활성 메뉴**: 회색 배경 칩으로 강조.
- **사이드바 활성 항목**: 좌측 보라 바 + 보라 텍스트.
- **드롭다운 메뉴**(지점 셀렉트 등): 클릭 시 패널 펼침, 외부 클릭으로 닫힘.

---

## 로딩(Loading) · 스피너 · 스켈레톤

- **버튼 로딩**: 스피너로 라벨 대체, 클릭 차단.
- **페이지 로딩**: 중앙 스피너 또는 스켈레톤 박스.
- **목록 lazy load**: 마지막 행 아래 작은 스피너.

---

## 빈 화면(Empty State) · 에러 화면

- **빈 화면**: 일러스트/아이콘 + "○○이 없어요" + 액션 버튼(만들기 등).
- **404/403**: 일러스트 + 안내 문구 + "홈으로" 버튼.
- **500**: "일시적 오류" + "다시 시도" 버튼.

---

## 알럿(Alert) · 헬퍼 텍스트(Helper Text)

- **인풋 헬퍼**: 입력란 아래 회색 작은 텍스트(11px), 입력 예시·제약 안내.
- **에러 헬퍼**: 같은 위치에서 빨간색으로 변경.
- **시스템 알럿(브라우저 confirm 대용)**: 모달 다이얼로그 패턴 따름.

---

## 단위 표기

| 데이터 종류 | 단위 |
|---|---|
| 금액 | 원 (예: 1,236,500원) |
| 횟수 | 회 / 건 (예: 1,432건) |
| 인원수 | 명 (예: 617명) |
| 비율 | % (예: 12%) |
| 시간(짧음) | 초 (예: 15초) |
| 시간(보통) | 분 (예: 8분) |
| 시간(전체) | 시간 |
| 온도 | 도 (°C) |
| 거리 | km / m |

## 비교/증감 표시

- "지난 7일 대비 +N%" 형태 뱃지.
- 상승=초록 + ↑, 하락=빨강 + ↓, 동일=회색 + −.

## 타이포그래피

- **폰트**: Pretendard 우선, 폴백 Inter.
- **타이틀(페이지)**: 30px Black, 자간 -0.6.
- **섹션 헤더**: 20px Bold.
- **본문**: 14px Regular.
- **캡션·헬퍼**: 11~12px Regular.

---

## 접근성(A11y)

- 모든 인터랙티브 요소에 키보드 탭 이동 가능.
- 모달 열림 시 포커스 트랩, 닫힘 시 트리거 버튼으로 포커스 복귀.
- 색상만으로 정보 전달 금지 (아이콘 또는 텍스트 병행).
- 폼 에러는 aria-describedby로 헬퍼 텍스트 연결.

---

## 사용 우선순위 (LLM 지시)

생성되는 기능 명세의 FE/BE 할일 작성 시 위 규칙을 **무조건 우선 적용**합니다.
- 화면에서 토스트가 감지되면 → "토스트 알림 표시 — fade-in 후 3초 뒤 fade-out" 그대로 사용
- 모달이 감지되면 → 위 "닫기 동작 3종"을 반드시 모두 포함
- 비활성 필드 → "읽기 전용으로 표시" 명시
- 인사이트 알럿 → 보라 배경 + ✨ 스파클 + 자동 생성 문구
- 단위는 위 표 그대로 사용

본 디자인시스템과 화면이 충돌하면 **화면(실제 UI)을 우선**하되, 모호한 부분은 본 정의로 결정합니다.
